# System Synthesis: The Solarpunk Architecture
**Date:** 2025-12-17
**Status:** Finalized

After 5 rounds of the "Philosopher Council" and a "CIA Sabotage Audit", we have crystallized the architecture of the Solarpunk Node.

## Core Philosophy: "Solarpunk with Teeth"

We realized that "Nice" systems are vulnerable to failure and sabotage. We have built a system that is **Anti-Fragile** and **Anti-Bureaucratic**.

### 1. The Death of the Ledger (Anti-Accounting)
**Problem:** Tracking every tomato turns mutual aid into a database of debt ("Reputation Capitalism"). It also allows "Haggle Sabotage".
**Solution:**
*   **Macro-Only Tracking**: We only track *Bulk* flows (Soup Kitchens, Barn Raisings).
*   **The "One Tomato" Rule**: If you give one tomato, it is invisible to the system. It is just a gift.
*   **Anonymous Gifts**: The right to give without credit.

### 2. Do-ocracy vs. Bureaucracy (Anti-Stalling)
**Problem:** "Silence Weight" and "Consensus" allow bad-faith actors to filibuster via inaction.
**Solution:**
*   **Tiered Decisions**: Constitutional changes need Consensus. Action (planting a garden) is **Do-ocracy**.
*   **Rough Consensus Override**: "You cannot block with words; only with code/alternatives."
*   **Participation Reflection (Pruning)**: Instead of waiting for the silent, we ask them: *"Do you still want to be here?"* Voluntary pruning keeps groups potent.

### 3. Molecular Stewardship (Anti-Scale)
**Problem:** Governance at scale fails.
**Solution:**
*   **Fractal Molecules**: Everything is a group of ~3-5 users (Molecules).
*   **Stewardship**: Groups exist to *steward* a resource (Heap). If they stop stewarding, the Counter-Power agent suggests dissolving/pruning.
*   **Osmosis**: Abundance automatically spills over to neighbors. Hoarding is impossible by protocol.

### 4. Conversational Excavation (Anti-Invisible Labor)
**Problem:** Care work is ignored.
**Solution:**
*   **The Interview**: The Agent talks to you. *"Did you rest?"*
*   **Oral History**: These chats become the community's memory.

### 5. Productive Condensation (Pro-Complexity)
**Problem:** "Flatness" destroys complexity. We need "Battery Nerds" to congregate to build microchips.
**Solution:**
*   **The Guild Protocol**: We distinguish between **Condensation** (Good) and **Accumulation** (Bad).
*   **The Metric**: A "Warlord" is defined not by how much they *have*, but by how much they *withhold*.
*   **High Stock + High Flow** = **Guild** (Celebrated).
*   **High Stock + Low Flow** = **Warlord** (Pruned).
